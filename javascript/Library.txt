💡 Library
API (Application Programming Interface), 라이브러리 (Library), 프레임워크 (Framework)는 모두 소프트웨어 개발에서 사용되는 용어다


Framework -> Library -> API (Application Programming Interface)
Framework 부품으로 Library 가 존재하고 API 는 Library 의 서브집합이다


1. 프레임워크
    
    단순하게 틀에 잡힌 업무 방식  (어떤 식으로 하는지에 대한 정해진 약속과 규범 ) 으로 `개발자들이 애플리케이션을 개발하는 데 사용되는 구조를 제공`한다. 프레임워크는 `일련의 규칙과 구조를 정의하고, 개발자가 애플리케이션을 작성할 때 이러한 규칙과 구조를 따르도록 한다.`
    

1. 라이브러리
    
    `개발자들이 자주 사용하는 코드 집합`이다. 라이브러리는 `여러 기능을 제공`하며, 이를 사용하여 개발자들은 `복잡한 기능을 더 쉽게 구현할 수 있다.` 라이브러리는 `여러 개발자가 공유하여 사용할 수 있는 소프트웨어 코드 모음`이다.
    
2. API 
    
    외부에 오픈된 것을 API 라고한다 Application Programming Interface
    
    API란 내가 만들 응용프로그램에서 사용 할 수 있도록 기타 다른 응용프로그램들을 제어 할 수 있게 하는 것을 말한다.
    

요약하면, API는 소프트웨어 컴포넌트 간의 인터페이스를 정의하는 것이고, 라이브러리는 개발자들이 자주 사용하는 코드 집합이며, 프레임워크는 애플리케이션을 개발하는 데 사용되는 구조를 제공하는 것이다.

WEB API - 라이브러리인데, WEB을 통해서. (전체 중 공개된 부분만)
대기업 또는 대규모 그룹에서 개인이 개발하기 어려운 서비스를 구축하여 대중에 서비스 하는 것
